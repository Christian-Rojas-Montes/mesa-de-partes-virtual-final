# Notificaciones, citas y recojo

El módulo existente produce notificaciones internas estructuradas para registro, recepción, observación, subsanación, derivación, atención, aprobación, rechazo, citas, originales, recojo, entrega y finalización. El correo es opcional y se sanitiza para excluir archivos y datos sensibles.

Las citas conservan fecha, rango horario, oficina, área, motivo, instrucciones, límite y estado. Una reprogramación mantiene trazabilidad. El recojo registra disponibilidad, oficina, validación de identidad, quien entrega y recibe, fecha y observación, sin almacenar copia del documento de identidad si no es necesaria.

El solicitante solo ve sus comunicaciones; el personal consulta el historial según rol. Los enlaces por correo conducen al sistema autenticado.
