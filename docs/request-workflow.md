# Flujo de solicitudes

1. El visitante consulta el catálogo y el detalle del trámite.
2. El solicitante se autentica, selecciona trámite y variante y declara datos de elegibilidad.
3. El servidor verifica actividad, ventana, modalidad, variante y prerrequisitos.
4. Se renderizan y validan campos y requisitos configurados mediante listas blancas.
5. El usuario revisa el resumen y confirma; el monto se obtiene del servidor.
6. Una transacción registra solicitud, respuestas, documentos, snapshot, estado inicial, historial, auditoría y notificación.
7. Se genera el código de seguimiento y comienza la atención.

Digital se considera presentado en el sistema; híbrido queda pendiente de originales; presencial puede generar preinscripción si está habilitada y su presentación oficial ocurre en Mesa de Partes. Observaciones, subsanaciones, derivaciones, citas, respuesta, recojo y entrega agregan eventos sin alterar el historial anterior.
